## 1. 用户

**简介**：在平台注册的自然人账号；承载登录凭证、公开资料、角色与是否被封禁。匿名游客不落库，不属于本实体。

**边界**：

- **拥有**：账号、密码凭证、个人展示字段、角色、账号状态。
- **不包含**：JWT Token 串本身（可无状态不落库）；第三方 OAuth；积分等级。
- **关联**：创建贴吧、发帖、评论、点赞、发起关注、接收通知、发起举报；管理员同属本表，用角色区分。

**主要属性**（逻辑名）：


| 属性     | 说明                    |
| ------ | --------------------- |
| 主键 ID  | 系统内用户唯一标识             |
| 登录账号   | 唯一，用于登录               |
| 密码摘要   | 哈希存储，不可明文             |
| 昵称     | 展示用                   |
| 头像 URL | 可选                    |
| 个人简介   | 可选                    |
| 角色     | 枚举：普通用户 / 平台管理员       |
| 账号状态   | 枚举：正常 / 封禁（或配合封禁截止时间） |
| 封禁截止时间 | 可选；空表示未封禁或永久策略由产品约定   |
| 创建时间   | 注册时间                  |
| 更新时间   | 资料等变更                 |

## 2. 贴吧

**简介**：用户创建的讨论版块容器；名称全局唯一，下有帖子列表。

**边界**：

- **拥有**：名称、展示信息、创建者引用。
- **不包含**：帖子正文（属帖子实体）；关注数等可通过关联统计，可不冗余存。
- **关联**：创建者 → 用户；帖子一对多；可被关注关系引用。

**主要属性**：


| 属性       | 说明      |
| -------- | ------- |
| 主键 ID    |         |
| 名称       | 全局唯一    |
| 简介       | 可选      |
| 封面 URL   | 可选      |
| 创建者用户 ID | 外键 → 用户 |
| 创建时间     |         |
| 更新时间     |         |

## 3. 帖子

**简介**：用户在某一贴吧下发布的主题，含标题与正文；可被评论、点赞、举报、检索。帖子采用单一可见状态流转（待审核/可见/审核拒绝/已隐藏）。

**边界**：

- **拥有**：标题、正文、所属贴吧、作者、时间戳、可见状态、软删除状态，以及热榜所需计数字段。
- **不包含**：评论列表（另一实体）；点赞明细（另一实体）；热榜排名（派生数据）。
- **关联**：贴吧、作者用户；评论、点赞、通知、举报审计可引用帖子。

**主要属性**：


| 属性        | 说明                  |
| --------- | ------------------- |
| 主键 ID     |                     |
| 贴吧 ID     | 外键 → 贴吧             |
| 作者用户 ID   | 外键 → 用户             |
| 标题        |                     |
| 正文        | 大文本                 |
| 可见状态      | 枚举：待审核 / 可见 / 审核拒绝 / 已隐藏；普通用户仅可见“可见且未删除”内容 |
| 审核人用户 ID  | 可选；管理员审核人           |
| 审核时间      | 可选；审核动作发生时间         |
| 审核备注      | 可选；驳回原因等            |
| 点赞数       | 必填；热榜计算输入            |
| 评论数       | 必填；热榜计算输入            |
| 浏览量       | 必填；热榜计算输入            |
| 创建时间      |                     |
| 更新时间      |                     |
| 删除标记或删除时间 | 软删除字段；支持「管理员删帖」与作者自删语义 |

## 4. 评论

**简介**：用户对帖子发表的回复；支持楼中楼结构。评论可被举报并可由管理员软删除/隐藏。

**边界**：

- **拥有**：所属帖子、作者、正文、根评论 ID、父评论 ID、时间、软删除与可见状态。
- **不包含**：点赞（需求未要求评论点赞）。
- **关联**：帖子、用户；可触发「被评论」类站内通知；可被举报审计记录引用。

**主要属性**：


| 属性        | 说明                  |
| --------- | ------------------- |
| 主键 ID     | 评论唯一标识              |
| 帖子 ID     | 外键 → 帖子             |
| 作者用户 ID   | 外键 → 用户             |
| 正文        | 评论内容                |
| 根评论 ID    | 楼中楼根节点；顶级评论可设为自身 ID |
| 父评论 ID    | 直接父评论；顶级评论为空        |
| 回复目标用户 ID | 可选，便于展示“回复某用户”      |
| 可见状态      | 枚举：可见 / 不可见（被管理员处理后可隐藏） |
| 创建时间      |                     |
| 更新时间      |                     |
| 删除标记或删除时间 | 软删除字段               |

## 5. 点赞

**简介**：用户对内容的点赞关系；支持点赞帖子与评论。当前业务不设计取消点赞（只增不删）。

**边界**：

- **支持目标类型**：帖子、评论。
- **操作模型**：仅点赞，不提供取消点赞；同一用户对同一目标最多一条记录（幂等）。
- **关联**：用户、帖子/评论；可触发「被点赞」通知。

**主要属性**：


| 属性    | 说明                            |
| ----- | ----------------------------- |
| 主键 ID | 或使用 (用户 ID, 目标类型, 目标 ID) 联合唯一 |
| 用户 ID | 外键 → 用户                       |
| 目标类型  | 枚举：帖子 / 评论                    |
| 目标 ID | 帖子 ID 或 评论 ID，与目标类型一致         |
| 创建时间  | 点赞时间                          |

## 6. 关注关系

**简介**：登录用户关注的对象，分两类：关注某个贴吧、关注某个用户。

**边界**：

- **一条记录表示一种关注**；用「目标类型 + 目标 ID」区分贴吧与用户，避免两张表也可接受，本总览按**统一关注关系**描述。
- **状态模型**：通过 `状态` 字段表示关注中/已取消；取消关注采用状态变更，不做物理删除。
- **不包含**：互关、粉丝分组等社交扩展。
- **关联**：关注者 → 用户；目标为贴吧 ID 或用户 ID（由类型区分）。

**主要属性**：


| 属性       | 说明                           |
| -------- | ---------------------------- |
| 主键 ID    | 或 (关注者 ID, 目标类型, 目标 ID) 联合唯一 |
| 关注者用户 ID | 外键 → 用户                      |
| 目标类型     | 枚举：贴吧 / 用户                   |
| 目标 ID    | 贴吧 ID 或用户 ID，与类型一致           |
| 状态       | 枚举：关注中 / 已取消                 |
| 创建时间     |                              |
| 更新时间     | 最近一次关注状态变化时间                 |

## 7. 站内通知

**简介**：系统或用户行为产生的站内消息（被评论、被点赞、系统通知等）；用户主动拉取列表并标记已读，无实时推送通道实体。

**边界**：

- **拥有**：接收人、触发人、通知类型、已读状态、展示文案、跳转目标信息。
- **不包含**：WebSocket 会话、邮件、短信。
- **关联**：接收人 → 用户；可关联帖子 ID、评论 ID、操作者用户 ID 等便于跳转。
- **读取约定**：通知列表按创建时间倒序读取（`created_at desc`）。

**主要属性**：


| 属性         | 说明                    |
| ---------- | --------------------- |
| 主键 ID      |                       |
| 接收用户 ID    | 外键 → 用户               |
| 触发用户 ID    | 可选；系统通知可为空            |
| 通知类型       | 枚举：被评论 / 被点赞 / 系统 / 等 |
| 标题或摘要      | 可选，便于列表展示             |
| 正文或扩展 JSON | 可选，存跳转参数或业务补充信息       |
| 目标类型       | 可选；枚举：帖子 / 评论         |
| 目标 ID      | 可选；与目标类型一致            |
| 是否已读       |                       |
| 已读时间       | 可选；首次已读时间             |
| 创建时间       |                       |
| 更新时间       |                       |

## 8. 内容举报审计

**简介**：记录从“用户举报”到“管理员处理”的全过程，作为举报审查与追踪依据。毕业设计场景下一张表承接举报与处理审计。

**边界**：

- **拥有**：举报人、对象类型、对象 ID、举报原因、处理状态、处理人、处理动作、处理备注等必要审计信息。
- **不包含**：高并发异步工单流转；本项目使用单表同步处理即可演示。
- **关联**：举报人/处理人均关联用户；对象关联帖子或评论；处理动作会驱动帖子/评论的可见状态或软删除状态变化。

**主要属性**：


| 属性         | 说明 |
| ---------- | ---- |
| 主键 ID      | |
| 举报人用户 ID   | 外键 → 用户 |
| 目标类型       | 枚举：帖子 / 评论 |
| 目标 ID      | 被举报对象 ID，与目标类型一致 |
| 举报原因       | 文本 |
| 举报补充说明     | 可选；用户补充材料 |
| 处理状态       | 枚举：待处理 / 已通过 / 已驳回 / 已执行 |
| 处理动作       | 枚举：忽略 / 隐藏内容 / 软删除内容 / 封禁用户（可扩展） |
| 处理备注       | 可选；管理员判定说明 |
| 处理人用户 ID   | 可选；外键 → 用户（管理员） |
| 处理时间       | 可选 |
| 创建时间       | 举报创建时间 |
| 更新时间       | |

## 派生与非实体说明（防边界混淆）


| 名称             | 说明                                                                |
| -------------- | ----------------------------------------------------------------- |
| **热榜排行**       | 由定时任务按固定公式计算，结果存 **Redis**（如有序集合）；**不是**与帖子平级的领域表实体，帖子上的统计字段仅为输入。 |
| **Feed 时间线**   | 基于「关注关系 + 帖子」的**查询视图**（方案 A），不单独建 Feed 表。                         |
| **搜索**         | 对贴吧、帖子的检索能力，不产生独立业务实体。                                            |
| **会话 / Token** | JWT 无状态时可无表；若做 Token 黑名单可另表，本总览不列为核心实体。                           |
