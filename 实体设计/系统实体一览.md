# 系统实体一览

本文档汇总本贴吧系统的**领域实体**：名称、简介、边界（职责与关系范围）、主要属性。用于需求对齐与后续表结构设计；**不表示已实现**。

**后续拆分约定**：每个实体在 `实体设计/` 下对应一个 Markdown 文件，文件名与下表「实体文件」列一致，内文可展开校验规则、状态机、索引建议等。

## 1. 用户

**简介**：在平台注册的自然人账号；承载登录凭证、公开资料、角色与是否被封禁。匿名游客不落库，不属于本实体。

**边界**：

- **拥有**：账号、密码凭证、个人展示字段、角色、账号状态。
- **不包含**：JWT Token 串本身（可无状态不落库）；第三方 OAuth；积分等级。
- **关联**：创建贴吧、发帖、评论、点赞、发起关注、接收通知、发起举报；管理员同属本表，用角色区分。

**主要属性**（逻辑名）：


| 属性     | 说明                    |
| ------ | --------------------- |
| 主键 ID  | 系统内用户唯一标识             |
| 登录账号   | 唯一，用于登录               |
| 密码摘要   | 哈希存储，不可明文             |
| 昵称     | 展示用                   |
| 头像 URL | 可选                    |
| 个人简介   | 可选                    |
| 角色     | 枚举：普通用户 / 平台管理员       |
| 账号状态   | 枚举：正常 / 封禁（或配合封禁截止时间） |
| 封禁截止时间 | 可选；空表示未封禁或永久策略由产品约定   |
| 创建时间   | 注册时间                  |
| 更新时间   | 资料等变更                 |

## 2. 贴吧

**简介**：用户创建的讨论版块容器；名称全局唯一，下有帖子列表。

**边界**：

- **拥有**：名称、展示信息、创建者引用。
- **不包含**：帖子正文（属帖子实体）；关注数等可通过关联统计，可不冗余存。
- **关联**：创建者 → 用户；帖子一对多；可被关注关系引用。

**主要属性**：


| 属性       | 说明      |
| -------- | ------- |
| 主键 ID    |         |
| 名称       | 全局唯一    |
| 简介       | 可选      |
| 封面 URL   | 可选      |
| 创建者用户 ID | 外键 → 用户 |
| 创建时间     |         |
| 更新时间     |         |


---

## 3. 帖子

**简介**：用户在某一贴吧下发布的主题，含标题与正文；可被评论、点赞、举报、检索。

**边界**：

- **拥有**：标题、正文、所属贴吧、作者、时间戳；删除策略（软删/硬删）实现阶段再定。
- **不包含**：评论列表（另一实体）；点赞明细（另一实体）；热榜排名（派生数据）。
- **关联**：贴吧、作者用户；评论、点赞、通知、举报可引用帖子。

**主要属性**：


| 属性        | 说明                  |
| --------- | ------------------- |
| 主键 ID     |                     |
| 贴吧 ID     | 外键 → 贴吧             |
| 作者用户 ID   | 外键 → 用户             |
| 标题        |                     |
| 正文        | 大文本                 |
| 创建时间      |                     |
| 更新时间      |                     |
| 删除标记或删除时间 | 可选，支持「管理员删帖」与作者自删语义 |


**扩展（非需求必填，与热榜相关）**：


| 属性              | 说明                           |
| --------------- | ---------------------------- |
| 点赞数 / 评论数 / 浏览量 | 可冗余计数便于热榜与列表展示；也可全量统计，实现时二选一 |


---

## 4. 评论

**简介**：用户对帖子发表的回复；需求以单层评论为主，楼中楼非必须。

**边界**：

- **拥有**：所属帖子、作者、正文、时间；删除语义与帖子类似。
- **不包含**：点赞（需求未要求评论点赞）；若不做楼中楼，则不包含父评论 ID。
- **关联**：帖子、用户；可触发「被评论」类站内通知；可被举报。

**主要属性**：


| 属性        | 说明                       |
| --------- | ------------------------ |
| 主键 ID     |                          |
| 帖子 ID     | 外键 → 帖子                  |
| 作者用户 ID   | 外键 → 用户                  |
| 正文        |                          |
| 父评论 ID    | 可选；边界为「仅一层回复」或「不做楼中楼则恒空」 |
| 创建时间      |                          |
| 更新时间      |                          |
| 删除标记或删除时间 | 可选                       |


---

## 5. 点赞

**简介**：用户对帖子「点赞 / 取消点赞」的持久化关系；同一用户对同一帖子至多一条有效记录。

**边界**：

- **只针对帖子**（与需求中「被点赞通知」一致）；不包含评论点赞。
- **关联**：用户、帖子；可触发「被点赞」通知。

**主要属性**：


| 属性    | 说明                      |
| ----- | ----------------------- |
| 主键 ID | 或使用 (用户 ID, 帖子 ID) 联合唯一 |
| 用户 ID | 外键 → 用户                 |
| 帖子 ID | 外键 → 帖子                 |
| 创建时间  | 点赞时间                    |


---

## 6. 关注关系

**简介**：登录用户关注的对象，分两类：关注某个贴吧、关注某个用户。

**边界**：

- **一条记录表示一种关注**；用「目标类型 + 目标 ID」区分贴吧与用户，避免两张表也可接受，本总览按**统一关注关系**描述。
- **不包含**：互关、粉丝分组等社交扩展。
- **关联**：关注者 → 用户；目标为贴吧 ID 或用户 ID（由类型区分）。

**主要属性**：


| 属性       | 说明                           |
| -------- | ---------------------------- |
| 主键 ID    | 或 (关注者 ID, 目标类型, 目标 ID) 联合唯一 |
| 关注者用户 ID | 外键 → 用户                      |
| 目标类型     | 枚举：贴吧 / 用户                   |
| 目标 ID    | 贴吧 ID 或用户 ID，与类型一致           |
| 创建时间     |                              |

## 7. 站内通知

**简介**：系统或用户行为产生的站内消息（被评论、被点赞、系统通知等）；用户主动拉取列表并标记已读，无实时推送通道实体。

**边界**：

- **拥有**：接收人、类型、已读状态、展示所需文案或关联 ID。
- **不包含**：WebSocket 会话、邮件、短信。
- **关联**：接收人 → 用户；可关联帖子 ID、评论 ID、操作者用户 ID 等便于跳转。

**主要属性**：


| 属性         | 说明                    |
| ---------- | --------------------- |
| 主键 ID      |                       |
| 接收用户 ID    | 外键 → 用户               |
| 通知类型       | 枚举：被评论 / 被点赞 / 系统 / 等 |
| 标题或摘要      | 可选，便于列表展示             |
| 正文或扩展 JSON | 可选，存跳转与参数             |
| 关联帖子 ID    | 可选                    |
| 关联评论 ID    | 可选                    |
| 触发用户 ID    | 可选，如点赞人               |
| 是否已读       |                       |
| 创建时间       |                       |


---

## 8. 内容举报

**简介**：普通用户对违规帖子或违规评论发起的举报单，供管理员处理。

**边界**：

- **拥有**：举报人、被举报对象类型与 ID、理由、处理状态（可选扩展）。
- **不包含**：管理员具体删帖操作日志（若不做审计表可不建实体）；删除帖子/评论本身由内容与服务完成，本实体仅记录举报诉求。

**主要属性**：


| 属性         | 说明                   |
| ---------- | -------------------- |
| 主键 ID      |                      |
| 举报人用户 ID   | 外键 → 用户              |
| 目标类型       | 枚举：帖子 / 评论           |
| 目标 ID      |                      |
| 举报理由       | 文本                   |
| 处理状态       | 枚举：待处理 / 已处理等，实现阶段细化 |
| 创建时间       |                      |
| 处理时间 / 处理人 | 可选                   |

## 派生与非实体说明（防边界混淆）


| 名称             | 说明                                                                |
| -------------- | ----------------------------------------------------------------- |
| **热榜排行**       | 由定时任务按固定公式计算，结果存 **Redis**（如有序集合）；**不是**与帖子平级的领域表实体，帖子上的统计字段仅为输入。 |
| **Feed 时间线**   | 基于「关注关系 + 帖子」的**查询视图**（方案 A），不单独建 Feed 表。                         |
| **搜索**         | 对贴吧、帖子的检索能力，不产生独立业务实体。                                            |
| **会话 / Token** | JWT 无状态时可无表；若做 Token 黑名单可另表，本总览不列为核心实体。                           |
